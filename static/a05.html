A05

1. Beschreibung des Datenmodells

In dieser Aufgabe wurde eine kleine Chat-Datenbank umgesetzt. Das Datenmodell besteht aus folgenden Tabellen:

  • user – registrierte Benutzer (id, username, email, …)
  • room – Chat-Räume (id, name, …)
  • room_member – Mitgliedschaften User↔Room (user_id, room_id, role, …)
  • message – Nachrichten in Räumen (id, room_id, user_id, text, sent_at)

Ein Raum enthält viele Nachrichten, und jede Nachricht gehört genau zu einem Benutzer und einem Raum.

4. Wichtige SQL-Abfragen & DB-Funktionen

Der Datenbankzugriff ist gekapselt in src/sqlite.ts. Beispiele wichtiger Funktionen:

getAllRooms()
getAllUsers()
getMessagesForRoom(roomId)

Beispiele verwendeter SQL-Abfragen:

SELECT id, name FROM room ORDER BY id;
SELECT id, username, email FROM user ORDER BY id;

SELECT m.id, m.text, m.room_id, m.user_id, u.username AS author
FROM message m
JOIN user u ON u.id = m.user_id
WHERE m.room_id = ?
ORDER BY m.id;
